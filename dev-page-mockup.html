Vibox v0.4 — 사이드바 리디자인

세온 · 방금 수정 · 4.1 KB · #vibox #design

배경

기존 단일 사이드바가 모든 권한이 한 메뉴에 섞여있어 파트너/매니저가 봐서는 안 되는 항목까지 노출됨. 드롭박스/VS Code 패턴을 차용해 레일 + 컨텍스트 메뉴로 분리.

구조

  • Rail (84px) — 홈 / 개발 / 내 박스 / 관리
  • ContextMenu (240px) — Rail 선택에 따라 동적 변경
  • Main (rest) — 페이지 본문

역할별 노출

  • Owner (세온): 4개 모두
  • Member: 홈 + 내 박스
  • Partner: 홈만 (자기 폴더 한정)
URL 기반 active state — Phase 4에서 /dev, /mybox, /admin로 라우트 재구성 예정.

다음 단계

Phase 4·5 완료 후, DevMenu의 폴더 항목을 실제 notes/ storage와 연동. 현재는 placeholder 4개(회의·프로젝트·마케팅·일기).

패널 너비Rail 84 + DevMenu 240 + 리스트 400 + 미리보기 = 넓고 시원함
장점본문 가독성 ↑ · 1280px 이하에서도 OK
단점3단 폴더 깊이는 DevMenu 안에서 처리해야 함 (펼침/접힘)
