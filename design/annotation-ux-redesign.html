vibox · 주석(annotation) UX 재설계

“동그라미 치고, 자막 고치고” 를
훨씬 쉽게 — 주석 다는 흐름 다시 짜기

기능(자막 교정·원·화살표·펜)은 다 있지만, 별도 ‘주석 모드’ + 영상 위 팝오버 구조라 한 번 다는 데 단계가 너무 많다. 핵심은 주석을 ‘그냥 코멘트’ 흐름 안으로 넣는 것.

지금

왜 불편한가 — 모드 + 팝오버, 6단계

1“주석” 버튼 클릭
(모드 진입·영상=캔버스)→ 2영상 위 툴바에서
도구 고르기→ 3영상에 드래그
그리기→ 4그 자리에
팝오버 등장→ 5팝오버에 입력
(자막=원본·수정 두 칸)→ 6제출
별도 모드 — “원 치고 코멘트”가 평소 코멘트 다는 거랑 완전히 다른 흐름. 모드를 켜고 꺼야 함.
팝오버가 또 뜸 — 입력칸이 두 군데(그림 위 팝오버 vs 하단 입력창)로 갈려 시선이 흩어짐.
도구 툴바가 영상 위 — 또 영상을 가리는 블럭이 생김.
자막 = 두 칸 — “브렌대 → 브랜드” 하나 고치는데 원본·수정 칸이 따로.
제안

주석을 ‘그냥 코멘트’에 통합 — 3단계

모드·팝오버를 없애고, 하단 코멘트 입력창에 그리기 도구를 넣는다. 그리면 그림은 프레임에 남고, 입력은 늘 같은 한 곳(입력창). 도구가 영상에서 빠져 영상도 깨끗.

1입력창에서 도구 클릭 (원·자막 등)→ 2프레임에 슥 그리기→ 3같은 입력창에 타이핑 → 등록
케이스 1 — 동그라미 치고 “여기 이렇게요”
00:03 / 00:10
원 화살표 펜 자막 수정 요청좋아요
00:03
여기 인물 얼굴에 빛이 너무 세요. 살짝 낮춰주세요

원을 누르고 프레임에 슥 그린 뒤, 같은 입력창에 그냥 코멘트. 팝오버 없음 · 모드 없음.

케이스 2 — 자막 “브렌대 → 브랜드”
자막 인식: ‘브렌대’
00:07 / 00:10
원 화살표 펜 자막 수정 요청좋아요
00:07
브렌대→브랜드

자막을 누르고 자막 위에 박스를 치면 OCR이 ‘브렌대’를 자동으로 칩으로 채워줌. 나는 고칠 말 한 줄만 타이핑. (원본·수정 두 칸 → 한 줄)

개선 6단계 → 3단계 개선 모드 없음 (평소 코멘트와 같은 흐름) 개선 팝오버 없음 (입력은 늘 한 곳) 개선 도구가 영상에서 빠짐 (영상 깨끗) 개선 자막 = 한 줄 (OCR 자동 채움)

반영 지점: FeedbackModal.tsx 의 AnnotationToolbar(영상 위)·AnnotationPopover 제거 → 하단 compose에 도구 + 인라인 입력으로 이전. 그리기(AnnotationDragLayer)·OCR(runOcr)·데이터 모델(annotation.ts)은 그대로 재활용. 여러 도형을 한 코멘트에 누적도 가능.
