vibox · 사람이 하는 검수(Review) 강화

프레임아이오는 사람 검수를
실제로 어떻게 하고, 우리는 어떻게

초점은 사람 — 클라이언트·매니저·편집자가 영상을 보고 그 순간에 피드백을 남기고, 주고받고, 새 버전으로 반영해 승인까지 가는 협업 검수 워크플로다. (자동/기술 QC는 이 문서 범위 밖.)

기준 Frame.io V4 (2024.10~, NAB 2025) 근거 vibox 실코드 + 1차 출처 대상 브랜치 beta
사람 검수 라이프사이클
클라/매니저영상 시청→ 그 프레임에 코멘트·주석→ 편집자반영 후 새 버전→ 다시 검수 (미해결 추적)→ 승인자승인 / 수정요청

A

Frame.io는 사람 검수를 실제로 어떻게 하나

핵심은 “정확한 위치에 + 정확한 의도를 + 흐르는 워크플로로”. 네 축이다.

① 코멘트 · 주석 · 협업 (리뷰어가 피드백을 남기는 방식)

그 프레임에 정확히

  • 프레임 정확 코멘트 — 일시정지 → 입력 → Send. 타임코드에 버블 + 우측 카드. 타임스탬프 표시/숨김 토글
  • 범위(range) 코멘트 — I/O 키 또는 플레이바 노란 bracket 드래그로 구간 지정. 카드 클릭 시 그 구간만 루프 (씬·BGM·자막 지속 피드백)
  • 주석(그리기) — arrow·rect·line + 자유 펜 + 색상 + undo/redo. 0~1 정규화 좌표 JSON으로 저장(해상도 독립). 평소 숨김, 그 코멘트 볼 때만 프레임 위 오버레이
  • 앵커 코멘트 — 재생 중 화면 임의 지점을 클릭/탭해 그 자리에 핀 고정

주고받기 + 정리

  • @멘션(자동완성 + 알림으로 바로 호출) · 답글 스레드 · 이모지 반응(코멘트 없이 동의/승인) · #해시태그 · 코멘트당 첨부 6개
  • 내부 vs 외부 — 내부 코멘트(자물쇠)는 외부 리뷰어에게 안 보임. 팀 논의와 클라 피드백 분리
  • 패널 — 정렬(타임코드/최신/작성자/완료) · 필터(주석·첨부·반응·안읽음·사람) · 본문 검색
  • 완료 추적 — Mark As Complete(=해결) 로 코멘트 체크오프, 미해결 vs 완료 추적 · 코멘트 딥링크 · CSV/XML/JSON 내보내기

② 버전 · 비교 (편집자가 반영하고 리뷰어가 다시 보는 방식)

버전 스택

  • 새 버전을 기존 위로 드래그해 수직으로 쌓음(폴더 불필요, 타입 혼합 허용). 기본은 최신 버전 표시
  • 상단 버전 번호 클릭으로 과거 버전 탐색, “Manage version stack”에서 순서 재배열

버전 비교 4모드

  • Side-by-side — 좌우 배치, 줌·재생 linked(동시 리뷰) / Unlink 토글
  • 오버레이 슬라이더 — 두 컷을 겹쳐 드러내며 정밀 비교
  • 픽셀 diff — 오른쪽이 흑백 + 변경 픽셀 하이라이트
  • 비교 중 코멘트 — 각 버전 헤더 클릭 → 올바른 버전을 겨냥해 코멘트

③ 승인 워크플로 + 게스트 리뷰 (결정과 흐름)

승인 — 역할 분리 + 명시적 결정

  • Reviewer vs Approver 분리 — Reviewer는 코멘트·마크업(완료 표시 가능, 필수는 아님), Approver는 반드시 결정을 내려야 다음 단계로
  • 결정 3종 — Approve(완료) / Approved with changes(사소 수정, 재승인 불필요) / Needs Work(새 버전 업로드 → 다시 승인 라운드)
  • 마감 자동 리마인더 — 마감 72h / 24h / 당일 자동 이메일
  • (네이티브는 status 메타데이터 + 자동 Collection으로도 구성)

게스트 리뷰 (클라이언트)

  • 계정 없이 링크로 시청 · 코멘트할 때만 이름/이메일(세션 한정 저장)
  • Share 토글 — Comment / Download on·off, 버전 표시 제어(모든 버전 vs 최신만), 자막 표시
  • 활동 추적 — Opened / Viewed / Commented / Download 를 이름·시각과 함께 피드로
  • 보안(고급, 엔터프라이즈) — 비번·만료, 허용명단만(포워딩 차단), 세션·포렌식 워터마크

④ 실시간 협업

Presence = 지금 같은 영상을 보는 사람을 아바타로 표시 + 코멘트 빠른 동기화·알림. (피그마식 라이브 공유 커서·동기 재생은 Frame.io 공식엔 미확인 — 그건 Dropbox Replay·SyncSketch의 영역.)

B

경쟁사 — 사람 검수의 차별점

시장은 다섯으로 수렴한다 — ① 리뷰어 무제한·무가입 ② 코멘트→태스크/투두 ③ 버전 넘어가도 미해결 유지 ④ 검수 진행 대시보드(누가 끝/미완) ⑤ 라이브 동시검수.

C

vibox 현황 — 사람 검수 관점

vibox는 개인이 영상 보고 코멘트 남기는 “검수 뷰어” 자체는 이미 거의 완성이다. 진짜 공백은 그 위의 협업·워크플로 층이다.

이미 있음 중복 제안 금지

검수 뷰어 (탄탄)

  • 프레임 정확 타임코드 코멘트 · 상태색 마커(미해결/AI/좋아요/해결)
  • 프레임 스텝(실측 fps + J/L/←→) · 좌우 코멘트 네비([/]) · 형제 영상 이동(K/J)
  • 주석/그리기 — bbox(자막교정) + arrow·ellipse·pen, 0~1 정규화, OCR 자막 추출
  • 답글 스레드 · 카테고리(6종)·Kind(feedback/praise/approve) 자동감지+override
  • 화질 토글(원본↔HLS 프록시) · 전체화면 몰입 · 순화 편집 · 정렬/필터 · 해결 마크

공유 · 게스트 · 승인

  • 승인 & 버전 핀 — kind="approve" + 파일명 v## 추출, 매니저 승인/취소
  • 공유 링크(preview/full · allowComments/Download · includeFeedback)
  • 게스트 검수 — 무가입·이름만. 모바일 세로=숏폼 몰입 뷰어, 데스크톱/가로=FeedbackModal
  • 클라별 코멘트 격리(같은 영상 여러 클라 공유 시 분리) · 공개/비공개 전환
  • 시청 추적 — 9초 비콘, 시청률·완료 (누가 봤는지)

부분  미완

  • 클라별 격리 — 컬럼만 nullable, share→client 매핑(Phase 1.5) 미완
  • 비번/만료 — 신규 링크 생성 UI에서 빠짐(레거시만)

없음  사람 검수 협업·워크플로 공백

  • 범위(in-out) 코멘트 — 단일 프레임만 (주석에 startMs/endMs 있음 → 재활용 가능)
  • @멘션 + 알림 (push 인프라 pushSubscriptions는 보유) · 이모지 반응
  • 코멘트 → 태스크(투두) 체크오프 (편집자가 코멘트별 처리완료)
  • 버전 스택 · 버전 간 비교(side-by-side) — 지금은 파일명 태그뿐
  • 버전 넘어가도 미해결 코멘트 carry (V1 미해결을 V2에서 추적)
  • 검수 진행 대시보드 (누가 검수 끝/미완·미해결 N건)
  • 승인 역할 분리·명시적 “수정요청” 결정 — 지금은 ‘승인’만, Needs-Work/조건부승인·Reviewer/Approver 없음
  • 마감 기반 리마인더 · presence/읽음 표시 · 라이브 동시검수 · 코멘트 내보내기
D

우리는 어떻게 강화할까 — 로드맵

원칙: FeedbackModal + comments 스키마 + 공유링크 위에 추가형으로. annotation.startMs/endMs, pushSubscriptions, clientVideos.status·resolvedAt·kind="approve" 는 이미 깔린 레일 — 재활용 우선.

Phase 1코멘트·협업 즉시 강화 (기존 위에 노출·연결)
1-1. 범위(in-out) 코멘트난이도 중가치 높
스키마 0 — annotation에 startMs/endMs 이미 존재. I/O 키로 구간 잡고 코멘트, 카드 클릭 시 그 구간만 루프. BGM·자막 지속·씬 단위 피드백에 바로 실효.
1-2. @멘션 + 알림난이도 중가치 높
pushSubscriptions 보유. 코멘트 @이름 파싱 → 해당 매니저/파트너/클라에 web push. “이 부분 확인해주세요”가 즉시 도달 — 검수가 핑퐁이 됨.
1-3. 이모지 반응난이도 낮가치 중
comment_reactions(commentId, userId, emoji) 테이블(IF NOT EXISTS) + 고정 셋. 코멘트 없이 동의/승인 표현 → 코멘트 노이즈↓, 가벼운 합의.
1-4. 코멘트 → 태스크 체크오프난이도 중가치 중~높
편집자(파트너)가 코멘트별 체크박스로 “처리 완료”. resolvedAt UI화 또는 taskDone 플래그. “미해결만 보기”와 합쳐 편집자의 수정 체크리스트가 됨.
1-5. 코멘트 내보내기(CSV/EDL)난이도 낮가치 중
/api/comments?format=csv|edl. 타임코드+카테고리+본문+해결상태. EDL은 크릿/Premiere 마커 임포트의 징검다리(Phase 3 포석).
Phase 2버전·승인 워크플로 격상 (개인 검수 → 팀 검수)
2-1. 검수 진행 대시보드난이도 중가치 높★ Wipster 빈칸
프로젝트/폴더 단위 “리뷰어별 시청·미해결 N건·승인 여부” 롤업. clientVideos.status·shareViews·resolvedAt·kind="approve" 집계 — 데이터는 이미 다 있음, 집계 뷰만. 매니저가 “어느 클라가 아직 안 봤나”를 한 화면에.
2-2. 버전 넘어가도 미해결 코멘트 carry난이도 중가치 높★ Replay 패턴
같은 폴더 v## 시리즈에서 V1 미해결을 V2 검수 화면 “이전 버전 미해결” 섹션으로. versionTag + filePath 형제 조회. 새 버전이 옛 지적을 반영했는지 추적 — 검수가 “끝났는지” 판단 가능.
2-3. 승인 워크플로 격상난이도 중가치 높
지금은 ‘승인’만. Reviewer/Approver 역할 + 명시적 결정 3종(승인/조건부승인/수정요청) 추가. “수정요청”은 자동으로 미해결 carry(2-2)와 묶임. (선택) 마감일 + 리마인더는 비봇 알림에 얹기.
2-4. 버전 비교(side-by-side)난이도 중~높가치 중
같은 폴더 두 영상 좌우 + 재생·줌 linked. 오버레이/픽셀 diff는 후순위. FeedbackModal을 2-up 그리드로, HLS 두 인스턴스 동기 seek.
Phase 3실시간·심화 (해자)
3-1. presence · 읽음 표시난이도 중가치 중~높
“지금 N명이 이 버전 보는 중” · 코멘트 읽음 표시(가벼운 폴링/SSE). 게스트 링크에서도 신뢰감. 라이브 동시검수의 전초.
3-2. 라이브 동시검수(같이 보기)난이도 높가치 높★ 큰 베팅
동기 재생 + 같은 화면을 매니저·클라가 동시에(Replay/SyncSketch 영역). presence 수요 확인 후. 원격 클라 미팅에 강력.
3-3. 모바일·클라 검수 UX 심화난이도 중가치 중
앵커 코멘트(임의 지점 핀)·핀치줌·세이프존 가이드를 ShortformReview에. 클라가 폰으로 먼저 본다는 현실에 맞춤.
3-4. AI 코멘트 요약 (검수 보조)난이도 낮~중가치 중
“이 버전 피드백 3줄 요약 + 중복 병합” 버튼(Claude 1콜). 사람 검수를 대체가 아니라 정리해주는 보조. 비봇 audit DNA와 일치.
E

더 나은 방안 — 모방 이상의 차별화

Frame.io를 베끼는 것 이상으로, vibox 강점을 사람 검수에 레버리지하는 방향.

영상제작사 도그푸딩(비모 1호)매니저↔파트너↔클라 역할·격리 보유한국어·국내 워크플로R2 egress 0(무제한 시청)크릿 자체 NLE비봇 능동 운영 문화

E-1. 검수 진행 대시보드 + 코멘트→태스크 (데이터 이미 보유)

Wipster·Replay가 셀링하는 “누가 끝/미완·미해결 N건”을 vibox는 신규 인프라 0로 만들 수 있다(status·resolvedAt·approve 집계). 비봇 “능동 운영자”(미해결 D+N 알림·카톡 운영방 도달)와 묶으면 검수가 알아서 굴러가는 운영이 된다 — Frame.io엔 없는 결합.

E-2. 클라이언트 입장에서 “세상에서 제일 쉬운 검수”

Frame.io도 클라에겐 여전히 어렵다. vibox는 무가입 + 모바일 세로 몰입(재생=깨끗/정지=등장) + 한국어로 “링크 받고 폰으로 보다가 톡 치듯 코멘트”를 극단까지. 클라 검수 전환율이 곧 제품력.

E-3. 검수 → 편집 타임라인(크릿) 엔드투엔드

Frame.io 최강 해자(코멘트가 Premiere 타임라인 마커로)를 Adobe 종속 없이 자체 NLE로 복제. vibox(검수)+크릿(편집)+비모ERP(프로젝트) 삼각이 닫힌 루프. 1-5 EDL export가 그 시작.

E-4. 모바일 세로 몰입 검수 심화

Frame.io조차 전용 세로 모드 근거 없음. vibox ShortformReview가 정확히 그 빈칸 — 앵커 코멘트·핀치줌·세이프존을 모바일에서 견고히 = 경쟁사 미보유 영역. 숏폼 시대 도그푸딩 차별점.

E-5. “개인 검수”가 아니라 “팀 검수” 워크플로

vibox는 이미 매니저·파트너·클라 역할과 격리를 갖췄다. 여기에 승인 역할 분리(2-3) + 대시보드(2-1) + carry(2-2)를 얹으면, 단순 코멘트 도구가 아니라 “검수 라운드”가 돌아가는 워크플로가 된다 — 국내 영상사 실제 검수 흐름(클라 1차→수정→2차→OK)에 정확히 맞음.

▶

결론 — 다음 작업 우선순위 (사람 검수)

  1. [즉시·1순위] 범위 코멘트(1-1) + @멘션 알림(1-2) + 이모지 반응(1-3) — 코멘트 협업의 즉시 체감. annotation startMs/endMs·pushSubscriptions 재활용으로 스키마 거의 0.
  2. [팀 검수 핵심] 검수 진행 대시보드(2-1) + 버전 carry(2-2) — 데이터는 이미 다 있고 집계/UI만. 매니저가 “누가 검수했나·뭐가 미해결인가”를 한눈에.
  3. [승인] 승인 워크플로 격상(2-3) — ‘승인’만 있는 현재에 ‘수정요청’ 명시 결정 + 역할 추가. 검수가 “라운드”로 굴러감.
  4. [보조] 코멘트→태스크(1-4) + 코멘트 요약(3-4) + 내보내기(1-5).
  5. [심화·야심] 버전 비교(2-4) → presence(3-1) → 라이브 동시검수(3-2) → 크릿 연결.
핵심 통찰: vibox는 “개인이 코멘트 남기는 검수 뷰어”는 이미 강하다. 다음 베팅은 그 위의 협업·워크플로 층 — ① 코멘트 협업(범위·멘션·반응) ② 팀 검수(대시보드·버전 carry·승인 라운드). 둘 다 이미 깔린 데이터/인프라를 꺼내 엮는 작업이라 가성비가 높다.
