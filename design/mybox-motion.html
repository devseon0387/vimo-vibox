My box — 폴더 열기 모션 & 로티 전략

"즐겁고 세련된"의 핵심은 절제된 모션. 일상 인터랙션은 CSS로 빠르고 부드럽게, 로티는 저빈도·고감성 '모먼트'에만. 클릭해서 직접 느껴보세요.

빠름 150–250ms 이징 ease-out / 스프링 절제 무채색·작게 CSS 미세 인터랙션 Lottie 모먼트만

1폴더 열기 애니메이션 — 두 가지 결

셰브론 회전 + 폴더 아이콘 열림(morph) + 하위 폴더가 높이 전환과 함께 위에서 차례로(stagger) 등장. 같은 동작, 다른 성격.

A · 절제 220ms · ease-out · 정직
B · 스프링 360ms · overshoot · 생기

추천: A 절제가 vibox의 차분한 정체성에 맞음. B는 살짝 더 즐겁지만 자주 보면 과할 수 있어 '가끔 보는 곳'에 어울림.

2즐겨찾기 별 — 팝 + 반짝

별을 누르면 톡 튀어오르며(spring scale) 작은 반짝임이 퍼짐. 0.4초·무채색. 이 정도 미세 모먼트는 CSS로 충분히 즐겁습니다.

진행중_프로젝트

한 단계 더 화려하게 가려면 이 자리에 Lottie 반짝 버스트를 끼울 수도 있음(아래 3번).

3로티는 여기에 — '모먼트'

폴더 열기 같은 매 순간이 아니라, 저빈도·감정적 순간에 로티를 한 스푼. 일러스트 특유의 따뜻함이 살아납니다.

LOTTIE 자리
업로드 완료 · 승인
체크마크 버스트 — 끝났다는 만족감 (예시 모션, 실제 = Lottie .json)
빈 상태
"폴더가 없습니다" 대신 친근한 일러스트 (이 자리 = Lottie)

인코딩 · AI 검수 진행 — 살아있는 로딩

위 3개는 CSS 예시 모션(실제론 해당 자리에 Lottie .json 에셋을 끼움). 로티 비용: 라이브러리 ~100–250KB + 에셋. 그래서 전역 미세 인터랙션엔 부적합, 이런 1~5곳 모먼트에만. 원하시면 실제 Lottie 에셋으로 바로 교체해 드립니다.

정리 — 모션 언어 = CSS 우선(빠름·이징·절제·무채색), Lottie는 모먼트 시즈닝. 폴더 열기 = CSS(A 절제 추천). 즐겨찾기 = CSS 팝(+선택적 Lottie 반짝). Lottie 모먼트 후보 = 빈 상태·업로드 완료·승인 체크·인코딩/AI 검수 로딩.
결(A/B) 고르시면 실제 앱(PartnerContextPanel 트리)에 CSS 전환을 넣고, 원하시는 모먼트만 Lottie 에셋을 끼웁니다.
