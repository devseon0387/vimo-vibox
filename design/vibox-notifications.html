비박스 알림 시스템

토스트·에러 메시지·다이얼로그·업로드 도크의 실제 패턴 정리. 모든 에러는 lib/human-error.ts 한 곳에서 친화 한국어로 변환됩니다(서버 raw 영문 노출 0).

1 토스트 2 에러 메시지 카탈로그 3 확인·삭제 다이얼로그 4 입력 다이얼로그 5 업로드 도크

1토스트 (Toast)

화면 하단 중앙에 스프링으로 등장하는 흰 알약형 알림. 종류별 컬러 아이콘 원(성공 초록·에러 빨강·정보 파랑). 본문 + 선택적 액션 버튼(주황) + 닫기.

영상이 업로드됐어요
이 공유 링크는 더 이상 열 수 없어요. 보낸 분께 새 링크를 요청해주세요.
이미 추가되어 있어요
파일 3개를 휴지통으로 옮겼어요

성공/정보는 짧은 사실, 에러는 "왜 + 다음 행동"까지. 액션 버튼(실행 취소 등)은 주황 텍스트로 절제.

2에러 메시지 카탈로그

서버 raw 영문 → 친화 한국어. 같은 에러도 맥락(컨텍스트)에 따라 다른 안내. 아래는 human-error.ts 실제 매핑.

서버 raw (전)비박스 안내 (후)

공유 링크 — 외부 클라이언트가 보는 면 share

revoked / expired이 공유 링크는 더 이상 열 수 없어요. 보낸 분께 새 링크를 요청해주세요.
password required / wrong password비밀번호가 필요하거나 올바르지 않아요. 보낸 분께 확인해주세요.
comments not allowed이 영상에는 의견을 남길 수 없도록 설정돼 있어요. 보낸 분께 문의해주세요.

권한 — 같은 "forbidden"도 맥락별로 forbidden

forbidden (library-write)자료실은 매니저·멤버만 올릴 수 있어요. 관리자에게 문의하세요.
forbidden (delete)이 항목을 지울 권한이 없어요. 본인이 올린 파일만 삭제 가능합니다.
forbidden (comment)댓글 권한이 없어요. 매니저에게 문의하세요.
unauthorized / 401로그인 세션이 만료됐어요. 새로고침 후 다시 로그인해주세요.

파일·휴지통 delete · move · rename · trash

not found (trash-restore)이미 영구 삭제됐거나 다른 사람이 복원한 항목이에요.
already exists (rename)같은 이름이 이미 있어요. 다른 이름을 사용해주세요.
invalid path / invalid name사용할 수 없는 경로·이름이에요. 특수문자(/ \ : * ? " < > |)는 빼주세요.
reserved path시스템 예약 폴더에는 작업할 수 없어요.

업로드·용량·네트워크 upload · quota · network

quota exceeded (personal-write)내 박스 용량이 부족해요. 관리자에게 요청하거나 오래된 파일을 정리하세요.
chunk count mismatch업로드 청크가 누락됐어요. 다시 시도해주세요.
network / fetch / timeout네트워크 오류가 발생했어요. 잠시 후 다시 시도해주세요.
aborted (upload)업로드가 취소됐어요.

폴백 (매칭 안 됨)

이미 한국어 메시지그대로 통과 (서버가 친화 메시지를 보낸 경우)
(빈 값)알 수 없는 오류가 발생했어요. 잠시 후 다시 시도해주세요.

3확인·삭제 다이얼로그 (Confirm)

파괴적 동작(삭제·제거·회수)은 danger 변형(빨강 원 + 빨강 버튼)으로 한 번 더 확인. 일반 확인은 무채색.
